# VA HOME v5.6 — порядок оновлення

1. Завантажте вміст основного архіву на GitHub поверх поточної версії сайту.
2. У Supabase → SQL Editor виконайте весь файл `SUPABASE_MIGRATION_V5_6_CUSTOMER_TRUST.sql` один раз.
3. У Supabase → Edge Functions створіть або оновіть функцію `submit-review` кодом із `supabase/functions/submit-review/index.ts`.
4. Для `submit-review` залиште **Verify JWT with legacy secret = OFF**. Функція сама безпечно перевіряє необов'язкову сесію покупця.
5. За бажанням додайте secret `REVIEW_RATE_LIMIT_SECRET` із довгим випадковим значенням. Без нього функція теж працює, але окремий secret посилює fingerprint rate limit.
6. Перевірте: вхід/вихід із кабінету, додавання й видалення wishlist, повторне замовлення, відправлення відгуку та його модерацію в адмінці.

`create-order` у цьому релізі не змінювався — повторно завантажувати його не потрібно.

## Що змінилося

- Кабінет більше не показує одночасно форму входу і профіль.
- Старі замовлення прив'язуються тільки після підтвердження email.
- Wishlist приватний, доступний лише власнику; серце додано в каталог і на сторінки ароматів.
- Відгуки надсилаються тільки через Edge Function із серверною валідацією та rate limit.
- Позначка «Перевірена покупка» визначається сервером за оплаченим/відправленим/виконаним замовленням, включно з ароматами в Discovery Set.
- Оптимізовано відкладене декодування карток зображень і social SEO для основних сторінок.
- Футер, контакти, доставка, оферта й політика конфіденційності містять узгоджені реквізити та контакти продавця.
- Checkout ущільнено в адаптивну двоколонкову форму з окремими блоками контактів і доставки/оплати.
